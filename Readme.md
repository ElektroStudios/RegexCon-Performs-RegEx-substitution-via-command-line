## 👋 Introduction

**RegexCon** is a lightweight, standalone command-line utility that brings the full power of the .NET Regular Expression engine directly to the Windows console. It evaluates an input string against a defined RegEx pattern and returns the formatted substitution to the standard output, making advanced text processing instantly accessible from the command line.

## 💡 Motivation

**RegexCon** was built for one specific purpose: bringing the full power of the .NET Regular Expression engine to environments that are otherwise completely blind to it. 

The project was born out of pure necessity. While working with a third-party application written in a limited scripting language, there was no native way to handle advanced RegEx substitutions. However, because that application allowed executing external programs, **RegexCon** became the missing link—handling the heavy-duty string manipulation outside the host environment and returning the exact output needed.

Whether you pipe data into it inside a rigid Windows batch file, use it within a `for-loop` for massive file renaming tasks, or drop it into automation software that lacks proper pattern matching, this utility forces full RegEx support onto any environment capable of spawning a process.

##### ⚡ The Real Question
###### What do you do when a third-party application lacks the core text processing capabilities you need? You can either spend hours writing fragile workarounds within its limited environment, or you can delegate the task to a compiled external binary that you know will handle the logic perfectly.

> [!IMPORTANT]
> **RegexCon** executes patterns strictly via the .NET Regular Expression engine. While standard expressions are highly compatible across modern environments, specific implementation details vary by programming language. For example, named capture groups follow a different syntax in PHP compared to .NET. Users are responsible for ensuring their patterns conform to .NET specifications.

## ⚙️ Usage

### Syntax
```cmd
RegexCon.exe /I=(input) /E=(expression) /S=(substitution) [/M=(true|false)]
```

### Available Switches

| Long Switch | Short Switch | Expected Value Type | Description |
| :--- | :---: | :---: | :--- |
| `/Input` | `/I` | String | The input string to be processed. |
| `/Expression` | `/E` | String | The regular expression pattern. |
| `/Substitution` | `/S` | String | The substitution or replacement string. |
| `/MatchCase` | `/M` | Boolean (`True` / `False`) | Enables case-sensitive matching. |
| `/?` | — | — | Displays the help menu. |

### Usage Examples

| Scenario | Command |
| :--- | :--- |
| **Regular string substitution** | `RegexCon.exe /I="Hello World" /E="^(.+)\s(.+)$" /S="$2 $1" /M=True` |
| **Named-group string substitution** | `RegexCon.exe /I="Hello World" /E="^(?<one>.+)\s(?<two>.+)$" /S="${two} ${one}"` |
